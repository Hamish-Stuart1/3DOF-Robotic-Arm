Software architecture

The system follows a layered architecture with a strict interface boundary between the PC application and the embedded firmware. Communication is over a UART serial link at 115,200 baud using a lightweight binary packet protocol.

PC application layer

  • WPF MVVM with data-binding to joint model
  • Real-time 2D kinematic viewport
  • SerialPort driver with framed packet protocol
  • Motion sequence editor with step playback
  • Torque warning and workspace limit UI

Firmware layer (C)

  • 50 Hz interrupt-driven control loop (TIM2)
  • PID position controller per joint
  • PWM output to servo drivers (TIM3/TIM4)
  • UART RX/TX with DMA for zero-blocking comms
  • Hardware E-stop via GPIO interrupt

Mechanical design

  • 3-DOF planar revolute kinematic chain
  • Link lengths: 120 mm / 90 mm / 60 mm
  • MG996R servos (J1/J2), SG90 micro (J3)
  • PLA/ABS printed structural links
  • Max reach: 270 mm, payload: ~150 g

Comms protocol

  • Binary framed packet: SOF / CMD / DATA / CRC8
  • Commands: JOINT_SET, HOME, ESTOP, STATUS_REQ
  • 20 ms telemetry heartbeat from firmware
  • PC-side ACK/NACK with retry logic

Firmware control loop

The firmware runs on an STM32F103 microcontroller. The main control task is triggered by a 50 Hz hardware timer interrupt, ensuring deterministic joint updates regardless of UART traffic. Each joint runs an independent PID position controller with configurable gains.

/* arm_controller.c - main control loop (50 Hz TIM2 ISR) */
#include "arm_controller.h"
#include "servo_driver.h"
#include "uart_protocol.h"

#define NUM_JOINTS     3
#define LOOP_FREQ_HZ   50
#define DT             (1.0f / LOOP_FREQ_HZ)

static JointState_t  joints[NUM_JOINTS];
static volatile bool estop_active = false;

/* TIM2 ISR - executes every 20 ms */
void TIM2_IRQHandler(void) {
    if (estop_active) { servo_disable_all(); return; }
    for (int i = 0; i < NUM_JOINTS; i++) {
        joints[i].pos_actual = encoder_read_deg(i);
        float error  = joints[i].pos_target - joints[i].pos_actual;
        joints[i].integral  += error * DT;
        float deriv   = (error - joints[i].prev_error) / DT;
        float output  = KP * error + KI * joints[i].integral + KD * deriv;
        joints[i].prev_error = error;
        output = clampf(output, JOINT_MIN[i], JOINT_MAX[i]);
        servo_write_pwm(i, deg_to_pwm(output));
    }
    uart_send_telemetry(joints, NUM_JOINTS);
    TIM2->SR &= ~TIM_SR_UIF;
}

/* GPIO EXTI - hardware E-stop line */
void EXTI0_IRQHandler(void) {
    estop_active = true;
    servo_disable_all();
    uart_send_event(EVT_ESTOP_ASSERTED);
    EXTI->PR = EXTI_PR_PR0;
}

Kinematic specification

The arm uses a planar 3R revolute chain. Forward kinematics are computed analytically using the Denavit-Hartenberg convention, giving the end-effector position as a function of the three joint angles θ₁, θ₂, θ₃, corresponding to base, shoulder and elbow.

Forward kinematics (end-effector XY position)
   x = L1·cos(θ1) + L2·cos(θ1+θ2) + L3·cos(θ1+θ2+θ3)
   y = L1·sin(θ1) + L2·sin(θ1+θ2) + L3·sin(θ1+θ2+θ3)
